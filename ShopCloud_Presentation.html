03 / 08 — Infrastructure
AWS Architecture Overview
Compute
⚡

Amazon EC2

t2.micro | Amazon Linux 2023

Apache web server hosting our ShopCloud website. Free tier eligible.

Database
🗃️

Amazon RDS

db.t2.micro | MySQL 8.0

Managed relational database. Stores products, categories and orders.

Storage
🪣

Amazon S3

Standard | us-east-1

Stores static HTML/CSS/JS files. Static website hosting enabled.

Networking
🔒

Amazon VPC

10.0.0.0/16 CIDR

Private network with public/private subnets, IGW, and security groups.

Internet → IGW → VPC → EC2 (public subnet) → RDS (private subnet) | S3 (static assets)

06 / 08 — How We Built It
8-Step Setup Process
1

AWS Account Setup

Created Free Tier account, set billing alarm, configured IAM.

2

VPC & Subnets

Built custom VPC 10.0.0.0/16 with public/private subnets and IGW.

3

EC2 Instance

Launched t2.micro Amazon Linux 2023. Configured security group rules.

4

Apache Web Server

SSH'd into EC2, installed Apache via dnf install httpd, enabled service.

5

S3 Bucket

Created public bucket, enabled static hosting, uploaded index.html.

6

RDS MySQL

Launched db.t2.micro in private subnet. Created products schema and seeded data.

7

Deploy Website

Uploaded ShopCloud HTML to EC2 /var/www/html/. Site went live!

8

GitHub & YouTube

Pushed all files to GitHub repo. Recorded and published video presentation.

07 / 08 — Reflection
Challenges & Lessons Learned
🔐

Security Group Misconfiguration

EC2 wasn't accessible from browser. Fix: Added inbound HTTP rule (port 80) to the security group.

🔗

EC2 → RDS Connection Failed

MySQL connection refused. Fix: Added port 3306 inbound rule in RDS-SG sourced from EC2 security group ID.

🪣

S3 Returned 403 Forbidden

Static site blocked. Fix: Disabled Block Public Access setting and applied the correct bucket policy JSON.

🔑

SSH Key Permission Error

PEM key rejected. Fix: Ran chmod 400 shopcloud-key.pem to restrict permissions on Mac/Linux.
